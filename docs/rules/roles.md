# 역할과 책임 (R&R) / 단계별 산출물

## 단계별 산출물 및 승인 주체

| 단계 | 산출물 | 작성 주체 | 승인 주체 |
|---|---|---|---|
| 1. 문서 (기획) | 요구사항 정의서 / 유저 스토리 | 기획자(사용자) | - |
| 2. 설계 | 기술 설계서(구조, API, DB 스키마 변경안, 영향범위) | Claude Code | 설계 승인자(사용자) |
| 3. 이슈 | 작업 단위로 쪼갠 이슈 (Definition of Done 포함) | Claude Code (승인된 설계 기반) | 사용자 (이슈 확정) |
| 4. PR | 이슈 1개당 PR 1개, 변경 파일/이유/테스트 결과 명시 | Claude Code | - (다음 단계에서 검증) |
| 5. 테스트 | 단위/통합/시나리오 테스트 결과 리포트 | Claude Code | - |
| 6. 검수 | 비전문가용 확인 가이드 + 실제 동작 확인 | Claude Code 작성, 사용자 실행 | 최종 의사결정자(사용자) |

## 역할

| 역할 | 담당 | 책임 |
|---|---|---|
| 기획자 | 사용자 | 요구사항 정의, 우선순위 결정 |
| 설계 승인자 | 사용자 | 기술 설계서 검토 및 승인/반려 |
| 개발 | Claude Code | 계획 제시 → 구현 → 테스트 → PR/검수 가이드 작성 |
| 검수자 | 사용자 | 검수 가이드에 따라 실제 동작 확인 |
| 최종 의사결정자 | 사용자 | 배포 여부, 범위 변경, 예외 승인 최종 결정 |
