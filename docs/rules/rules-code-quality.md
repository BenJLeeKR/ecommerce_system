# 코드 / 테스트 품질 기준

> 이 문서는 `CLAUDE.md` 절대 규칙 6, 8의 실행 지침이다.

## 완료(Definition of Done) 기준

신규 기능은 최소한 다음을 포함해야 "완료"로 간주한다.

- 정상 케이스 테스트
- 주요 실패/예외 케이스 테스트 (결제/재고는 동시성·경계값 케이스 포함)
- 기존 테스트 회귀(regression) 통과

## 머지 기준

- 테스트가 실패한 상태에서는 병합하지 않는다. "일단 머지하고 나중에 고친다"는 예외를 두지 않는다.
- 커버리지 수치보다 **핵심 시나리오(결제 실패, 재고 부족, 중복 주문 등) 커버 여부**를 우선한다.

## 코드 스타일

- 린트/포맷 규칙은 프로젝트 설정 파일 기준을 따르며, 임의로 규칙을 끄지 않는다.
- 규칙 예외가 필요하면 사유를 코드 주석 또는 PR 설명에 명시하고 별도로 보고한다.
