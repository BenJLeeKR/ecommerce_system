# 비전문가 검수 가이드 작성 양식

> 이 문서는 `CLAUDE.md` 절대 규칙 7의 실행 지침이다. 각 PR/기능 완료 시 아래 양식으로 검수 가이드를 작성한다.

## 양식

```markdown
### 1. 무엇이 바뀌었는가
(한두 문장, 비개발자 언어로)

### 2. 왜 바뀌었는가
(연결된 이슈 번호 / 기획 요구사항)

### 3. 어떻게 확인하는가
1. (클릭 순서, 입력값)
2. ...
기대 결과: (무엇이 보여야 정상인지)

### 4. 무엇이 위험한가
- 실패 시 영향 범위:
- 되돌리는 방법:

### 5. 테스트 결과 요약
- 통과/실패 여부:
- 실패 시 원인:
```

## 원칙

- 검수자는 이 가이드를 따라 실제로 동작을 확인한 뒤 승인/반려를 결정한다.
- 코드 diff만으로 승인하지 않는다.
- 클릭 순서/입력값은 코드 지식이 없어도 그대로 따라할 수 있는 수준으로 구체적으로 작성한다.
