# 배포 및 롤백 원칙

> 이 문서는 배포를 준비하거나 실행하는 작업을 할 때 참조한다.

## 원칙

- 운영 배포 전 스테이징(또는 이에 준하는) 환경에서 검증을 거친다.
- 모든 배포 단위는 **롤백 계획**을 사전에 명시한다 (이전 버전 재배포, 마이그레이션 되돌리기 등).
- 결제/주문/재고처럼 장애 시 금전적 손실로 직결되는 영역은 가능하면 기능 플래그(feature flag)로 즉시 비활성화할 수 있게 설계한다.

## 배포 전 체크리스트

- [ ] 스테이징 검증 완료
- [ ] 롤백 계획 명시 (이전 버전 재배포 방법 / 마이그레이션 down 스크립트)
- [ ] 민감 영역(결제/주문/재고) 변경 시 기능 플래그 적용 여부 확인
- [ ] 모니터링/알림 대상 확인 (배포 후 무엇을 지켜봐야 하는지)
