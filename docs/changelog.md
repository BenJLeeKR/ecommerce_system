# 규칙 문서 개정 이력 및 개정 원칙

## 개정 원칙

- 프로젝트 진행 중 원칙이 부족하거나 과도하다고 판단되면, 별도 논의를 거쳐 관련 문서를 개정한다.
- `CLAUDE.md`의 절대 규칙(1장)은 최종 의사결정자만 개정할 수 있으며, Claude Code가 임의로 해석을 완화하지 않는다.
- 세부 문서(docs/*)는 실무 진행 중 발견된 반복 이슈를 반영해 수시로 개정할 수 있다. 단, 절대 규칙과 충돌하는 개정은 불가하다.

## 이력

| 날짜 | 변경 내용 | 문서 |
|---|---|---|
| (최초 작성일) | 절대 규칙 8개 + 작업 원칙 최초 수립 | 전체 |
| (분리 작업일) | 단일 문서를 CLAUDE.md(핵심) + docs/*(세부) 구조로 분리 | 전체 |
