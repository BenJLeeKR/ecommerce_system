# 민감 영역 처리 세부 지침 (인증/권한/결제/주문/재고/개인정보)

> 이 문서는 `CLAUDE.md` 절대 규칙 5의 실행 지침이다. 인증/권한, 결제, 주문, 재고, 개인정보(PII) 처리와 관련된 작업을 할 때만 참조한다.

## 적용 대상

- 로그인/인증, 권한(Role/Permission) 체크 로직
- 결제 요청/승인/취소/환불 처리
- 주문 생성/변경/취소 처리
- 재고 차감/복구/동시성 처리
- 개인정보(이름, 연락처, 주소, 결제수단 등) 저장/조회/전송

## 필수 사전 보고 항목

위 대상에 대한 변경은 **PR 생성 전** 다음을 문서로 제시하고 승인을 받는다.

1. 변경 사유
2. 기존 로직과의 차이 (Before/After)
3. 실패 시 롤백 방법
4. 동시성/중복 처리 검토 결과 (예: 결제 중복 승인 방지, 재고 오버셀 방지 로직)

**사용자의 명시적 승인 없이는 이 영역에 대한 PR을 생성하지 않는다.** (초안·제안 문서 작성까지는 가능)

## 결제

- 실제 PG 연동 전 반드시 샌드박스/테스트 모드에서 검증한다.
- 결제 실패, 타임아웃, 중복 요청 등 예외 케이스를 테스트에 포함한다.

## 개인정보

- 로그, 에러 메시지, 커밋 메시지에 개인정보 원문을 남기지 않는다 (마스킹 처리).
- 개인정보를 다루는 신규 API/필드 추가 시, 최소한의 수집 원칙을 따르는지 확인 후 보고한다.

## 재고/주문 동시성

- 동시 요청 시 재고가 음수가 되거나 중복 주문이 발생하지 않는지 시나리오 테스트로 검증한다.
- 락(lock) 또는 원자적 연산을 사용하는 경우, 데드락 가능성을 함께 검토한다.
